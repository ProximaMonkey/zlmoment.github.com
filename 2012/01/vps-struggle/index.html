VPS折腾记

Sat 14 January 2012 by Zhaoyu Li

实在不爽虚拟主机的性能问题和每天几十封宕机警告，而且主机的一切环境都是虚拟主机商配置好的，没有自由发挥的余地。于是果断决定给博客找个好点的VPS，也为以后博客改版做准备。当然，还有VPN哈哈。

因为Linode VPS口碑还不错，而且支持ipv6，所以就选它了。折腾了整整一下午一晚上，终于把小博客搬了过来，简单总结一下整个过程以备忘。

SSH登陆

在Windows上，可以使用Putty来使用SSH。在Mac上，可以直接使用Terminal。命令很简单：

ssh username@server -p portNumber

后面的-p portNumber只在主机不是使用标准端口时需要加上，使用时将username和server替换成自己的用户名和地址就可以了。

安装LNMP

网上普遍说LNMP比LAMP性能要好，虽然我这个小博客也没多少流量，但无折腾不生活，趁年轻多折腾折腾。

可以选择自己一个一个配置，也可以选择LNMP一键安装包。当然折腾的前提是要稍微熟悉它，所以暂时用一键安装包了。官方给出了详细的安装步骤（见这里），这里就不再赘述。对于安装中遇到的各种问题，请看本文第五部分。

安装完后记住将Nginx和PHP升级到最新版，安装包中的不是最新版。

安装Wordpress

不知为何，按照LNMP官方教程安装VSFTPD总是不成功，于是放弃之安装了PureFTPd。Wordpress更新时需要FTP支持，所以先配置FTP（这不是必然的，是有一定原因的。）。PureFTPd是直接使用一键安装包中的集成版，非常方便。

./pureftpd.sh

然后根据提示输入相关信息即可。完成后访问http://hackecho.com/ftp即可进行配置，当然首先是给自己添加一个FTP账号了。

WordPress的安装比较简单，也就是下载、解压，然后访问即可自动安装。

wget http://cn.wordpress.org/wordpress-3.3.1-zh_CN.tar.gz

tar zxvf wordpress-3.3.1-zh_CN.tar.gz

安装完成后，将先前的博客中的内容导出，在新博客中导入即可。（在导入的过程中出现一个问题，新的系统不识别导出的xml文件，而xml文件是完全没有问题的。经过多次尝试无果，这里采用的方式是直接将原先的整个数据库搬了过来。但直接般库会出现很多问题，一定要仔细修复。）

配置伪静态

一切都完成后发现博客只有主页可以访问，而其余所有页面都是404。原来是在原先的系统中固定链接采用了自定义结构，而到这里没有对服务器进行伪静态规则配置，所以无法解析地址。尝试改回Wordpress的默认固定链接就可以了，但这会导致博客内所有n页面的地址都发生变化，弄不好谷歌百度会K站的。所以，还是还原回原来的配置吧。

主要参考了这两篇文章《Nginx的Rewrite配置》《Nginx下WordPress的Rewrite》，完成了伪静态配置。Nginx和Apache还是有区别的。

以下是我的.htaceess内容：

location / {
    index index.html index.php;
    if (-f $request_filename/index.html){
        rewrite (.*) $1/index.html break;
    }
    if (-f $request_filename/index.php){
        rewrite (.*) $1/index.php;
    }
    if (!-f $request_filename){
        rewrite (.*) /index.php;
    }
}

最后记得在Nginx配置文件中的server块中include进去，重启Nginx。

/etc/init.d/nginx restart

遇到的问题及解决办法

1、Wordpress无法更新（插件和系统）、无法下载安装插件

检查目录权限。一般文件夹为755，但我设置成755还是不可以，直接将wp-content文件加777后可以了。

chmod -R 777 wp-content

2、LNMP安装后出现502 Bad Gateway问题

这个问题在这篇帖子里有详细的叙述，但也许是人品问题，PHP无法编译和启动。后来经过无数次尝试，发现只要将VPS的系统安装为Ubuntu 10.04即可，我原先安装的是ubuntu 11.10。有相同问题的朋友可以试一下。

3、提示无法定位WordPress Content目录(wp-content)

在wp-config.php中加入如下代码：

if(is_admin()) {add_filter(‘filesystem_method’, create_function(‘$a’, ‘return “direct”;’ ));define( ‘FS_CHMOD_DIR’, 0751 );}

问题继续发现中，如果大家还有其它什么问题，欢迎一起讨论。
