Mac OS X上装MySQL for Python的辛酸经历

Sat 18 February 2012 by Zhaoyu Li

折腾了一下午，只为一个MySQL for Python...

记得寒假的时候在家就为装这个折腾了半天，遇到的是同样的问题，但后来稀里糊涂装好了，也没有做笔记。没想到今天又遇到了相同的问题，解决之后，特地记下解决方法。相信跟我同样问题的人很多，而网上却丝毫找不到相关的资料...

首先说一下安装环境：

操作系统：Mac OS X Lion

Python版本：2.7

MySQL for Python版本：1.2.3

问题：

编译MySQL for Python的源码时，出现如下错误提示：

unable to execute gcc-4.2: No such file or directory
error: command ‘gcc-4.2′ failed with exit status 1

然后第一反应当然是Google之，关于这个error的结果还真不少，但是，如果您正在读这篇文章，就请把Google或百度结果页面关掉吧。我可以非常肯定的告诉你，前几十页之内，没有一个搜索结果可以解决这个问题！

这个问题的原因，其实是与苹果系统的GCC编译器有关（其实苹果的gcc编译器叫做LLVM，还有一段非常的历史，我记得在某期的《程序员》杂志上看到过，有兴趣的读者可以看这篇文章《Mac OS X 背后的故事（八）三好学生Chris Lattner的LLVM编译工具链》。）关于我的一系列苦逼的修复过程我就不重复了，直接公布解决办法吧。

解决办法：

做连接gcc-4.2到命令llvm-gcc-4.2即可。（当然，这只是一种最笨的方法，更高明方法还没想到）

sudo ln -s /usr/bin/llvm-gcc-4.2 /usr/bin/gcc-4.2

这条命令的关键是：连接源一定要是llvm-gcc-4.2，不能是gcc，否则会出现更多搜都搜不到的错误！

然后运行编译命令：

python setup.py build
python setup.py install

安装成功：

Processing dependencies for MySQL-python==1.2.3  
Finished processing dependencies for MySQL-python==1.2.3
